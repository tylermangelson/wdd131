Unit 1

Variables and Constants

1.1 The Storage Locker (Variables)

Variables are containers for data. Use let when the value will change (like a score).

Project: Character Profile

Create variables for name, level, and health. Update them as the character "levels up."

1.2 The Locked Box (Constants)

Use const for values that never change (like BIRTH_YEAR or PI).

1.3 Data Types

Understand Strings ("text"), Numbers (10), and Booleans (true/false).

Unit 1 Synthesis

How does choosing between let and const help other programmers understand your intentions?

Unit 2

Conditionals

2.1 The Fork in the Road (If/Else)

If statements act as gates. "If the user is logged in, show the dashboard."

[Image of programming if-else flow chart]

Project: Password Validator

Write a script that checks if a password is longer than 8 characters.

2.2 Multiple Paths (Else If)

Use else if to handle more than two possible outcomes (e.g., Gold, Silver, or Bronze medals).

2.3 Logic Gates (&& and ||)

Combine conditions. "If it is raining AND I have an umbrella."

Unit 2 Synthesis

How do conditionals allow a program to feel "intelligent" to the user?

Unit 3

Functions

3.1 The Machine (Declaration)

A function is a reusable block of code. Define once, use many times.

3.2 Inputs (Parameters)

Project: Tip Calculator

Create a function that takes billAmount and tipPercent as inputs and returns the total.

3.3 Outputs (Return)

The return keyword sends a value back to where the function was called.

Unit 3 Synthesis

Why is "Don't Repeat Yourself" (DRY) the most important rule in programming?

Unit 4

Arrays

4.1 The Egg Carton (Indexes)

Arrays store lists of data. Remember: computers start counting at 0!

4.2 List Management

Use push() to add to the end and pop() to remove from the end.

4.3 Iteration (Loops)

Project: Grocery List

Create an array of items and use a for loop to print each item with "Buy: " in front of it.

Unit 4 Synthesis

How do arrays and loops work together to handle 1,000 items as easily as 1 item?

Unit 5

Objects

5.1 Key-Value Pairs

Objects group related data. Instead of 3 variables for a car, use one car object.

5.2 Object Methods

Methods are functions that belong to an object, representing things the object can "do."

5.3 Nested Data

Project: Social Media Post

Create an object for a "Post" that contains a username, content, and an array of comments.

Unit 5 Synthesis

How do objects allow us to model the real world (cars, users, products) in our code?

Unit 6

Events

6.1 Event Listeners

Listeners wait for something to happen (click, scroll, type) before running code.

6.2 Mouse & Click Events

Project: Color Switcher

Create a button that changes the background color of the page every time it is clicked.

6.3 Keyboard Events

Capture what keys the user presses to create shortcuts or game controls.

Unit 6 Synthesis

Why are events considered the "bridge" between the user and the logic of the app?

Unit 7

The DOM

7.1 Selecting Elements

The DOM is a tree of your HTML. Use querySelector to grab branches of that tree.

7.2 Modifying Elements

Change text with .innerText or change appearance with .style.

7.3 Creating Elements

Project: Dynamic To-Do List

Take text from an input box and create a new <li> element inside a list when a button is clicked.

Unit 7 Synthesis

How does JavaScript use the DOM to turn a static document into a living application?

Unit 8

Sorting

8.1 Logic of Comparison

Sorting involves comparing two items at a time and deciding which "wins" the higher spot.

8.2 The .sort() Method

JavaScript's default sort is alphabetical. To sort numbers, we use (a, b) => a - b.

8.3 Bubble Sort

Project: Leaderboard Sorter

Take an array of player objects {name: "X", score: 10} and sort them by highest score first.

Unit 8 Synthesis

When you look at a site like Amazon or Netflix, how are Units 4, 5, and 8 working together to show you results?
